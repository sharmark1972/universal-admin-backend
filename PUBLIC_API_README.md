# IJARCM Public API

This API provides access to published papers from the International Journal of Academic Research in Commerce and Management (IJARCM). It allows external websites to programmatically access and display papers from this journal.

## Base URL

```
https://your-domain.com/api/public
```

## Authentication

While the API works without authentication, using an API key is recommended for:
- Higher rate limits
- Better service quality
- Usage tracking

API keys can be generated by administrators through the admin panel at `/admin/api-keys`.

## Rate Limiting

- Default: 100 requests per minute per IP address
- Custom limits can be set per API key
- Rate limit headers are included in all responses:
  - `X-RateLimit-Limit`: Rate limit for the current period
  - `X-RateLimit-Remaining`: Remaining requests in the current period
  - `X-RateLimit-Reset`: Timestamp when the rate limit resets
  - `Retry-After`: Seconds to wait before retrying (when rate limited)

## Endpoints

### 1. Get Papers List

Retrieve a list of published papers with optional filtering and pagination.

**Endpoint:** `GET /papers`

**Parameters:**
- `query` (string, optional): Search term to filter papers by title, abstract, keywords, or author names
- `category` (string, optional): Filter papers by category
- `author` (string, optional): Filter papers by author name
- `dateFrom` (string, optional): Filter papers published after this date (YYYY-MM-DD format)
- `dateTo` (string, optional): Filter papers published before this date (YYYY-MM-DD format)
- `sortBy` (string, optional): Sort papers by field. Options: `relevance`, `date`, `downloads`, `title`. Default: `date`
- `sortOrder` (string, optional): Sort order. Options: `asc`, `desc`. Default: `desc`
- `page` (number, optional): Page number for pagination. Default: `1`
- `limit` (number, optional): Number of papers per page (max 50). Default: `20`
- `apiKey` (string, optional): Your API key for authentication

**Example Request:**
```bash
curl "https://your-domain.com/api/public/papers?query=machine learning&limit=10"
```

**Endpoint:** `GET /papers/{id}`

## Error Codes

- `400`: Bad Request - Invalid parameters
- `401`: Unauthorized - Invalid API key
- `403`: Forbidden - Insufficient permissions
- `404`: Not Found - Resource not found
- `429`: Too Many Requests - Rate limit exceeded
- `500`: Internal Server Error

## CORS

The API supports Cross-Origin Resource Sharing (CORS) for use in web applications. The following headers are included:

- `Access-Control-Allow-Origin`: `*`
- `Access-Control-Allow-Methods`: `GET, OPTIONS`
- `Access-Control-Allow-Headers`: `Content-Type, Authorization`
- `Access-Control-Max-Age`: `86400`

## Integration Examples

### JavaScript

```javascript
// Fetch all papers
fetch('https://your-domain.com/api/public/papers')
  .then(response => response.json())
  .then(data => console.log(data));

// Search papers with filters
fetch('https://your-domain.com/api/public/papers?query=machine learning&category=Computer Science&limit=10')
  .then(response => response.json())
  .then(data => console.log(data));

// Get specific paper
fetch('https://your-domain.com/api/public/papers/paper-id')
  .then(response => response.json())
  .then(data => console.log(data));

// With API key
fetch('https://your-domain.com/api/public/papers?apiKey=your-api-key-here')
  .then(response => response.json())
  .then(data => console.log(data));
```

### Python

```python
import requests

# Fetch all papers
response = requests.get('https://your-domain.com/api/public/papers')
data = response.json()
print(data)

# Search papers with filters
response = requests.get('https://your-domain.com/api/public/papers', params={
    'query': 'machine learning',
    'category': 'Computer Science',
    'limit': 10
})
data = response.json()
print(data)

# Get specific paper
response = requests.get('https://your-domain.com/api/public/papers/paper-id')
data = response.json()
print(data)

# With API key
response = requests.get('https://your-domain.com/api/public/papers', params={
    'apiKey': 'your-api-key-here'
})
data = response.json()
print(data)
```

## Best Practices

1. **Cache Responsibly**: Implement client-side caching to reduce API calls
2. **Handle Rate Limits**: Respect rate limit headers and implement backoff strategies
3. **Use Specific Queries**: Use specific parameters instead of fetching all data when possible
4. **Error Handling**: Implement proper error handling for all response codes
5. **Secure API Keys**: Never expose API keys in client-side code
6. **Pagination**: Use pagination for large datasets instead of fetching all records
